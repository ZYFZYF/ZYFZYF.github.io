w2-网络入侵和攻击

  • 入侵分为对系统的和对网络的
  • 网络扫描，用来对发起攻击的目标进行了解
    • 主机发现（IP/ICMP）
    • 端口扫描（nmap工具）
      • 逐个对一段端口或指定的端口进行扫描。通过扫描结果可以知道一台计算机上都提供了哪些服务，然后就可以通过所提供的这些服务的己知漏洞就可进行攻击
    • 操作系统检测（同上，一条命令nmap -O + IP 即可）
    • 漏洞扫描（Nessus扫描软件）
  • ZMAP的扫描有什么用呢（统计网络流量的信息？）
  • Dos攻击
    • 拒绝服务攻击Denial of Service
      • 攻击机制
        • 异常数据：利用协议的漏洞
          • Teardrop攻击：向被攻击者发送多个分片的IP包（IP分片数据包中包括该分片数据包属于哪个数据包以及在数据包中的位置等信息），某些操作系统收到含有重叠偏移的伪造分片数据包时将会出现系统崩溃、重启等现象
          • Ping of death：通过分片传输大于64k的包，导致系统崩溃
          • Land 攻击：采用了特别构造的TCP SYN数据包（通常用于开启一个新的连接），使目标机器开启一个源地址与目标地址均为自身IP地址的空连接，持续地自我应答，消耗系统资源直至崩溃。这种攻击方法与SYN洪泛攻击并不相同
        • 洪泛型：向目标发送大量数据包
  • DDos攻击
    • 分布式Dos
    • 新型DDoS攻击
      • 放大DDoS攻击
        • memcache是一套分布式的高速缓存系统，目前被许多网站使用以提升网站的访问速度，尤其对于一些大型的、需要频繁访问数据库的网站访问速度提升效果十分显著
        • UDP反射放大型DDOS攻击只需攻击者向Memcache服务器发送小字节请求，就可以引发服务器将数万倍的响应数据包发送给攻击者，形成灾难级的DDoS攻击（为什么是memcache这么遭难）
      • Mirai Botnet进行DDoS攻击
        • Botnet 是指采用一种或多种传播手段，将大量主机感染bot程序（僵尸程序）病毒，从而在控制者和被感染主机之间所形成的一个可一对多控制的网络
      • IPv6
  • 突然想到异常检测能不能不止用来检测入侵，如果能检测出上线的代码的bug就好了(:з」∠)
  • 蠕虫/病毒
  • 缓冲区溢出攻击
  • APT攻击
    • Advanced Persistent Threat：高级持续欺骗
    • 钓鱼邮件+office宏执行（这是啥）+植入木马+窃取信息

w3-研讨

w4-入侵检测概述

  • 一种动态的监控、预防或抵御系统入侵行为的安全机制
  • 通用入侵检测理论模型（Denning模型）
    • 主题（subjects）、对象（objects）、审计记录（audit records，就像日志一样吧）、简介（profiles这个翻译有点怪(:з」∠)，我感觉老师也是感觉翻译成啥都不合适所以才没写中文的）
    • Profiles里有活动的描述，包括统计度量和统计模型
      • 统计度量
        • 事件计数器：在一个时间段内满足某个属性的审计记录的数量
        • 间隔计数器：两个相关事件之间的时间间隔
        • 资源测量：一个时间段内某种动作所消耗的资源
      • 统计模型
        • 观察新的$x$是不是异常的
        • 好多模型…
    • 基于规则的模式匹配系统
  • 入侵检测技术的分类
    • 数据源：HIDS vs NIDS （host vs net）
      • 基于主机：系统日志、审计信息、系统调用、应用程序日志
        • 优点：
          • 可以直接访问动作的语义（更易于阻断攻击、更难绕过）
          • 能够对非网络威胁提供保护（这个是什么意思呢）
          • 对于加密行为有一定可见性
          • 性能扩展更容易
      • 基于网络：原始报文、流级流量信息、SNMP
        • 优点：
          • 单点部署可以检测很多系统，方便管理
          • 方便部署
          • 不必浪费端系统的生产资源
    • 分析方法：误用检测 vs 异常检测
      • 误用检测：
        • Misuse Detection，又称为基于特征的检测？
          • Snort规则（是一个开源的IDS）
        • 优点：有效地发现一致攻击
        • 缺点：未知攻击；定义变种也很困难
      • 异常检测
        • 优点：理论上可以检测未知攻击
        • 缺点：虚警率较高
      • 不是很理解这两个的区别…
    • 检测方式：实时 vs 非实时
    • 响应方式：主动 vs 被动
      • 主动：检测出入侵后，自动采取相关措施 IDPS就是检测+防护
      • 被动：仅告警

w7-开源IDS

  • Snort（是一只猪）
    • 三种运行模式：读取显示网络数据包、将数据包存到硬盘中、网络流量分析与检测
    • 四大模块
      • 数据包嗅探解码模块
      • 预处理模块
        • 从snort.conf中加载配置
        • 报文重组预处理器
          • frag3：IP分片重组和攻击检测
          • stream：维持TCP流状态，进行会话重组
        • 协议规范化预处理器
          • http_inspect：规范HTTP流（规划URI）
          • rpcDecode：规范调用
        • 异常检测预处理器
          • arpspoof：检测ARP欺骗（ARP是从MAC到IP的转换）
          • sfportscan：检测端口扫描
          • ARP（从IP到MAC地址的转换协议）欺骗
      • 检测模块
        • 检测头
          • 五种响应机制：Activation（报警并启动另外一个动态规则链）、Dynamic（由其它的规则包调用）、Alert（报警），Pass（忽略），Log（不报警但记录网络流量）
        • 检测规则
          • 有点没看懂具体格式，之后再看(:з」∠)
      • 报警/日志模块
  • Suricata（商标是狐獴）
    • 最突出的特点是多线程。它的多线程架构是独一无二的，因为它可以支持高性能的多核和多处理器系统。多线程设计的主要优点是它可以提高网络流量分析的速度和效率，还可以根据处理需求的位置帮助划分IDS / IPS工作负载
    • 自动协议检测：自动识别网络使用的协议，并应用一定的规则，不管你提供的端口是啥；一定程度上也防止了用户错误
  • Bro（商标是眼睛，为什么你这么特殊，就像XLNet一样(:з」∠)）
    • 最特殊的是Event Engine，根据监测到的网络报文跟踪连接的状态，并产生相应的事件
    • 有个规则解释器，所以可以写Bro语言？更加灵活
  • 性能比较
    • suricata的CPU利用率好低啊

w8-入侵检测标准化与测评

  • 标准化
    • 各个系统之间的协作、需要与其他系统交换信息，所以得有标准的交互接口
  • CIDF：Common Intrusion Detection Framework
    • 定义了一个通用的入侵检测框架以及各个组件之间通信的协议和API
    • gidos: generalized intrusion detection objects
  • CISL：Common Intrusion Specification Language
    • 这是什么语言
  • IDS协作
    • 分析：一个收集数据初步分析、一个深入分析产生结果
    • 互补：互相弥补对方的覆盖度，然后合并产生结果
    • 增强：互相纠错
    • 验证：用另一个来验证第一个的报警是否属实，其实和增强差不多
    • 调整：根据另一个的报警信息来调整监测阈值之类的？
    • 响应：根据另一个的告警自动响应
  • CIDF和IETF IDWG标准化的核心都是入侵告警描述语言，前者类似LISP语言，后者采用XML格式
  • 测评检测能力（其实就是一个二分类器）
    • AUC
    • EER：当FPR和FNR相等时的值，越小越好

放大DDos攻击

  • UDP不会握手，所以谁发送请求应答谁，所以伪造正常用户的IP地址，这样服务求就会向正常用户洪泛大量流量，使得正常用户无法正常工作（应答流量比接受流量的意思是：请求可以用很少的话解释，但是回复可能有很多东西）
  • 从放大器和正常用户的角度来看都无法识别到attacker，本质上还是原来协议设置的漏洞，让伪装变得这么简单
  • 该论文提出的是从扫描放大器的过程中挖掘attacker的信息（但是scanner要是也做了源地址伪造，不就一点办法都没有了吗）
  • 对于不同的scanner，hash一下返回的结果，这样根据被attack的服务器就知道这个scanner是有问题的
    • 多个scanner
    • 选择一部分服务器进行攻击
    • 这个置信度的计算暂时没有理解
  • 如何进行网络定位？如果知道它的位置到三个server的距离，画圆就知道了（即三边测量技术）
    • 距离用TTL，但是
      • 初始TTL是不知道的（无需知道，只要比较跳数的变化即可）
      • 路由变化
      • 网络拓扑未知
    • 验证了attacker是否与scanner使用了同一基础设施
  • 感觉想法和实验还是蛮有意思的(:з」∠)

网络流量异常检测研讨

  • ASTUTE
    • 网络异常的来源
      • 奇怪的行为（Dos、端口检测）
      • 错误配置or行为
      • 正常（大文件传输）
    • 动机
      • 能学到新的异常吗
      • 如何不需要学习什么是异常就能检测出异常
    • ASTUTE异常检测新方法（短时间不相关-交通均衡）
      • 无需训练
      • 很好的检测一种well-defined class of 异常
    • ASTUTE的特点是可以检测很多小流量，而Kalman和Wavelet检测的是少量的大流量
    • 均衡模型
      • 时间片
      • 流之间不相关、平均值为0、同分布
  • Mining Anomalies Using Traffic Feature Distributions
    • 通过流的特征分布来检测异常（例如Dos攻击目的地会集中分布在被攻击的流量上），之前的工作都是基于流量大小来判断的
    • 分布的区别就用熵，这里用的是样本熵？（意思是我们无法得知特征的准确分布，所以只能采样来获取一个度量）
    • 使用聚类来对异常进行分类
    • 主要过程是将entroy检测与volume检测进行对比

基于人工智能的异常检测

  • A Deep Learning Approach for Network Intrusion Detection System
    • 误用检测和异常检测的区别：
      • 误用检测：定义所有的异常行为，将所有其他行为定义为正常，所以理所当然地无法检测到未知的异常
      • 异常检测：定义正常行为，然后将其他所有行为定义为异常，关键在于定义主题正常活动的“活动简档”
      • 异常检测有两个困难的地方：
        • 一是特征选择
        • 二是labeled data 不可获得或者代价太高
    • 许多NIDS做了一个特征选择任务从训练数据中的提取了一个相关自己的特征来增加分类的效果
      • 特征不是越多越好吗
    • STL = self-taught learning
      • 两步，第一步是UFL，即非监督的特征提取工作，有很多方法例如AutoEncoder，RBM，K-Means；第二部是有监督的分类，这两部分的数据可以是来源不同的，但是也一定会有一定的相关性
      • Autoencoder中cost function是均方差+L2系数+KL散度，相当于用隐藏值的value来当提取出来的特征
    • PCA + SVM是个好思路
    • NSL-KDD相比较KDD Cup Dataset做了两点改进：
      • 去除了训练集和测试集中的无用/冗余记录
      • 把record分了难度？
    • 作者的第二步是用了Softmax 分类方法？（就是直接接一层Softmax层做多分类？也没法问题。。。毕竟要相信前面的工作已经提取出好的特征来了）
    • 用Autoencoder提取特征和直接一层隐藏层直接接分类的softmax的区别在哪儿？中间这个隐藏层不也相当于是特征提取了吗
      • 目的不一样：autoencoder的目的是提取最有用的信息来还原出所有的信息，而隐藏层的目的就是拟合label，所以会有微小的差别？
